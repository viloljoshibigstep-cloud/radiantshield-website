The Challenge

Healthcare Needs AI — But Can't Expose Patient Identity

Sharing unprocessed DICOM data with external vendors creates compliance risks, verification gaps, and manual overhead.

18

PHI Exposure Risk

HIPAA identifiers at risk in every raw DICOM transfer. Unprocessed imaging sent to third-party AI vendors leaks patient names, IDs, and dates — violating HIPAA, DPDP Act, and GDPR.

0%

No Independent Verification

Most hospitals cannot independently verify vendor anonymisation. There is no proof that PHI is removed before AI models process your data.

3–5×

Manual Workflow Overhead

IT teams script DICOM transfers per vendor, juggle proprietary APIs, and manually de-identify studies per modality — multiplying effort for every new integration.

The Solution

What is RadiantShield?

A secure data exchange layer between your PACS and external AI, tele-radiology, and research services.

DICOM Anonymisation

PS3.15-compliant metadata de-identification plus pixel-level PHI redaction via OCR/CV. Configurable profiles per workflow and modality.

Assurance Gate

Independent 39-tag PS3.15 validation on every outbound study. Blocks and alerts on any residual PHI. Zero-trust verification before data leaves.

Intelligent Routing

Declarative rules match studies by Modality, Body Part, and more. Each rule targets a specific AI vendor with tailored anonymisation profiles.

Reversible Restoration

UUID-based Alias ID per study. Deterministic re-identification from encrypted Secure Ledger. Lossless DICOM fidelity on return.

How It Works

Five-Step Secure Pipeline

Every action logged in an immutable audit trail — full traceability for HIPAA, DPDP Act, and GDPR.

01

Ingest

C-STORE, REST API, or batch from PACS / RIS

02

Anonymise

Metadata + pixel PHI removal. Assign Alias ID.

03

Validate

39-tag PS3.15 check + SHA-256 integrity hash.

04

Route

Forward via DICOMweb, REST, or C-STORE.

05

Return

Re-identify AI results & deliver back to PACS.

Core Capabilities

Built for Production Healthcare Workloads

Configurable Profiles

3 presets: Strict, AI-Friendly, Research. Per-tag actions: Keep, Remove, Replace, Offset. PS3.15 codes auto-populated.

Pixel-Level Redaction

OCR/CV detection of burned-in text. Image masking and redaction pipelines. Modality-aware processing for US, SC, NM.

Multi-Protocol Egress

DICOMweb STOW-RS for cloud PACS. REST API for proprietary AI endpoints. C-STORE SCU for traditional targets.

Encrypted PHI Vault

AES-256-GCM at rest and in transit. Client-controlled encryption keys. Append-only, auditable Secure Ledger.

Validation & Integrity

Post-anonymisation PHI removal verification. Restoration completeness checks. SHA-256 manifest hashes per study.

Monitoring & Alerting

Real-time throughput and latency metrics. Alerting on failures and SLA breaches. Admin dashboard with full visibility.
